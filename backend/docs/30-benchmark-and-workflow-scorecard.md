## Scorecard Parser Normalization

- `release:scorecard` now extracts the first complete top-level JSON payload from each subcommand instead of scanning backward for the last nested object
- `repo-hygiene` is now part of the `engineering_floor`, so deleted legacy paths cannot silently re-enter the official workspace surface
- `extensions-workflow` is now part of the `engineering_floor`, so task-level skill and MCP closure has to stay regression-backed
- `skill-compatibility` is now part of the `engineering_floor`, so Claude-style instruction bundles cannot silently regress back into untyped imported placeholders
- `real-task-completion` is now part of the `engineering_floor`, so SCC has to keep passing repo-real complex tasks rather than only synthetic or family-level scenarios
- `public_capability_parity` is now part of the `engineering_floor`, so public OpenCode / Claude Code / SWE-bench-style task shapes cannot silently regress
- `manual_artifact_audit` is now part of the `engineering_floor`, so automated green runs still have to survive evidence-based artifact review
- `practical_task_acceptance` is now part of the `engineering_floor`, so SCC has to keep passing real mixed user-task handling rather than only engineering-shaped scenarios
- `practical_manual_audit` is now part of the `engineering_floor`, so practical-task outputs have to stay ship-ready under explicit human review
- `workflow-scenarios` is explicitly unwrapped from the nested `workflow` key
- `breadth`, `flagship`, `general-complex`, `live-provider`, and `benchmark` now keep their full report payloads
- payload-shape mismatches are surfaced as `open_gap` parsing issues instead of silently collapsing to `0 / 0`
- default and local provider-enabled scorecards now write to separate report files so the default environment result is not overwritten by the ignored local env helper
- Postgres local-helper runs now also write to a separate profile report file, so the default scorecard keeps its public environment stance
- scorecard pre-cleaning now preserves `release-scorecard*.json` profile reports while still clearing transient smoke and historical workspace logs

## Remaining Scorecard Gaps

- Postgres-backed validation still requires explicit database environment setup
- provider timeout/retry/stress scorecard is richer than before, but still thinner than the scenario and benchmark evidence in default environments where live-provider execution is disabled
- browser validation now has both smoke and strict E2E layers; remaining browser work is deeper visual regression and longer-session operator coverage rather than missing mutation-flow automation
- context-history remains the residual benchmark-side cost center even though the current token target is met
- richer general-complex coverage now exists, but external rich-media chains such as `docs / ppt / mp4` remain intentionally out of scope for this stage
- dual-track skill compatibility is verified, but marketplace lifecycle depth, dependency resolution, and automatic MCP wiring remain intentionally out of scope for this stage
