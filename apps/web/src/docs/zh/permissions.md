# 权限审批

权限审批页定义的是“真实工具调用的边界”，不是回答风格偏好。

## 5 种审批模式

### Ask

默认最保守。工具只要涉及风险类别，就先问你。

### Read only

自动放行主机观察和工作区读取，但写入、Shell、联网和破坏性操作仍然需要你确认。

### Full access

全局允许全部风险类别，包括 destructive。只有在工作区可恢复、任务边界可信时才建议使用。

### Custom

允许你按风险类别逐项开关，适合在只读和完全访问之间做精细边界。

### Auto approval

只对选中的**非破坏性**风险类别启用自动审批。destructive 仍然不会被规则自动放开。

## 风险类别

- **host_observation**：查看进程、资源占用等主机状态
- **workspace_read**：读取文件、列目录、搜索代码
- **workspace_write**：编辑或创建本地文件
- **shell**：执行命令
- **network**：联网访问
- **destructive**：删除、覆盖、终止进程等高风险行为

## MCP 审批模式和 LLM 自动审批

- **MCP 审批模式**：只影响 MCP 工具，不会改写普通内置工具的风险边界
- **LLM 自动审批**：实验能力，只会在规则仍需审批时介入，而且最多批准非破坏性工具
