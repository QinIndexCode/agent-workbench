# 设置中心

设置中心不是附属页面，而是 SCC 的运行边界控制台。这里决定：

- 任务默认走哪个模型
- 工具调用如何审批
- 外部工具和消息入口是否可用
- 定时任务是否会自动运行
- 搜索是否能联网
- 个人偏好和本地存储行为如何表现

## 第一次配置建议顺序

如果你是第一次使用，建议按这个顺序完成：

1. 先在 **模型配置** 添加一个真实可用的模型 Provider。
2. 再到 **权限审批** 选择合适的审批模式，默认建议从只读开始。
3. 如果需要外部工具，再配置 **MCP**。
4. 如果需要外部消息入口，再配置 **集成**。
5. 如果需要重复任务，再配置 **定时任务**。
6. 如果任务需要联网证据，再配置 **网络搜索**。
7. 最后用 **偏好** 调整语言、主题和显示习惯。

## 7 个分区分别做什么

### 模型配置

负责连接模型服务、保存本地密钥、选择当前模型，以及配置失败时的回退 Provider。

### 权限审批

负责定义工具调用的风险边界。这里不是“是否信任模型”，而是“哪些行为允许自动执行，哪些行为必须让你确认”。

### MCP

负责把外部工具服务器接进 SCC。即使工具来自 MCP，它们仍会进入 SCC 的同一套审批和时间线。

### 集成

负责把 Discord、飞书、Slack、Telegram 或企业微信消息接成普通 SCC 任务。外部消息不会绕过文件夹、模型和审批策略。

### 定时任务

负责在 SCC 打开时按计划创建任务。它不是隐藏系统服务，也不会假装在应用关闭后继续运行。

### 网络搜索

负责配置内置 `web_search` 可使用的搜索来源。这里添加的是 Provider，不是给 Agent 直接开后门联网。

### 偏好

负责个人使用习惯和本地行为，不直接改变工具风险边界。

## 如何使用“查看说明”

设置页内的“查看说明”按钮会直接打开当前分区对应的手册，而不是把你丢回总目录再自己查。适合在以下情况下点击：

- 你不确定某个字段应该填什么
- 你想知道改动会影响哪些运行行为
- 你要给团队其他成员写一份操作说明
