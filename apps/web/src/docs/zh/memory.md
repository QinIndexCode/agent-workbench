# 记忆

记忆是 Agent Workbench 在多轮任务之间持续携带的背景上下文。

## 两份核心文档

- **USER.md**：你的长期偏好和行为约束
- **MEMORY.md**：当前文件夹对应的项目级长期记忆

## 结构化项目记忆

适合保存：

- 架构选择
- 技术栈说明
- 业务逻辑约束
- 仓库约定

## 上下文分层

Agent Workbench 会按生命周期拆分上下文：

- **系统核心层**：永久规则和安全边界，永不压缩
- **长期记忆层**：USER.md、项目 MEMORY.md、结构化项目记忆和已加载的可复用 Skill
- **近期会话层**：可审计摘要 + 最近对话和工具证据的原始滑动窗口
- **即时工作层**：当前轮、Knowledge/RAG 指针、附件、Known Files、审批状态和任务连续性

知识库检索结果和自动注入的 Knowledge Brief 属于即时工作层，只帮助下一步行动；除非你明确要求修改长期记忆，否则不会写回记忆。

每次请求模型前，运行时会预估上下文窗口。约 70% 时轻裁剪低价值原始内容，85% 时更新会话摘要，95% 时强制压缩大工具日志，保证当前轮仍能正常完成。

## 保持记忆干净

- 去掉一次性任务结果
- 让描述短而可复用
- 当 `MEMORY.md` 开始重复膨胀时，执行压缩
