# Task Management

任务是 Agent Workbench 的核心组织单元，每个任务对应一次独立的会话上下文。

## 创建任务

- 点击侧边栏的 **添加任务** 按钮创建新任务
- 新任务会出现在侧边栏的任务列表中
- 点击任务标题可切换当前会话

## 目标完成模式

当你希望 Agent 追求可验证的结果，而不是只给出简短回答时，可以输入 `/goal <需求>`。目标完成模式会运行更久，持续展示验收标准、工具进度、文件变更和验证状态，并在探索、实现、验证之间继续推进，直到任务完成，或因上限、权限、模型异常、无进展恢复或用户中断而暂停。

启动 `/goal` 一定会弹出确认框。请仔细阅读后果提醒：该模式可能消耗更多模型额度，反复读写文件、运行命令或访问网络。选择 **Full risk** 会全局允许 destructive 操作，例如删除、覆盖和终止进程，必须额外勾选确认后才能启动。

## 任务历史

- 所有任务会话会自动保存
- 侧边栏的任务列表支持滚动浏览
- 点击任务可查看完整的历史消息

## 任务执行流程

1. 用户发送消息
2. 系统组装上下文（历史消息 + 当前输入 + 附件内容）
3. 模型生成响应
4. 如响应包含工具调用，系统按权限策略处理
5. 工具执行结果返回模型，继续生成
6. 最终结果展示给用户
